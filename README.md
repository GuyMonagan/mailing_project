# mailing_project
Django-приложение для управления email-рассылками с системой ролей, статистикой отправки и расширенным интерфейсом.

---
## Основной функционал

-  Регистрация и аутентификация пользователей
-  Расширенный профиль с аватаром, телефоном и страной
-  Управление шаблонами сообщений
-  Добавление и редактирование получателей
-  Создание и планирование рассылок
-  Отправка писем вручную (через интерфейс)
-  Роли: пользователь / менеджер
-  Отчёты по отправкам
-  Просмотр статистики по каждой рассылке
-  Деактивация рассылки менеджером
-  Кэширование главной страницы
-  Админка Django

---
##  Стек технологий

- Python 3.11+
- Django 5.x
- PostgreSQL
- Poetry
- Bootstrap (CSS-фреймворк)
- Django Admin
- Django Messages Framework

##  Структура проекта


- `users/` — регистрация, вход, профиль, кастомная модель `User`
- `mailing/` — модели сообщений, рассылок, получателей, попыток отправки
- `config/` — глобальные настройки проекта
- `templates/` — все HTML-шаблоны
- `media/` — пользовательские аватары
- `.env.example` — пример конфигурации окружения

##  Быстрый запуск проекта

1. **Клонировать репозиторий:**

```
git clone https://github.com/GuyMonagan/mailings_project.git
```

2. Создать виртуальное окружение и активировать его:
```
python -m venv venv
source venv/bin/activate        # для Linux/macOS
# или
venv\Scripts\activate           # для Windows

```
или
```
poetry instal
```

3. Установить зависимости:
``` 
pip install -r requirements.txt
```

4. Создать файл .env на основе .env.example:
   
```
SECRET_KEY=ваш-секрет
DEBUG=True
EMAIL_HOST_USER=example@example.com
EMAIL_HOST_PASSWORD=your-password
```
   
5. Применить миграции:

```
python manage.py migrate

```
  
6. Создать суперпользователя:

```
python manage.py createsuperuser
```
7. Запустить сервер:
```
python manage.py runserver
```

## Роли

- Пользователь: видит только свои данные, может управлять рассылками.
- Менеджер: может просматривать всех пользователей, деактивировать чужие рассылки.
- Суперпользователь: полный доступ через Django Admin.

## Дополнительно

- Главная страница кэшируется на 15 минут 
- В отправке используется SMTP-сервер (настраивается в .env)
- Отчёт по рассылке доступен в деталях рассылки и списке

## SMTP

- Для отправки писем необходимо подключить почтовый сервер.
- Используется стандартный django.core.mail.backends.smtp.EmailBackend. 
- Настройки в .env.
